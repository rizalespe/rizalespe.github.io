Sebelum mulai

Kamus istilah, biar nggak bingung

Skim aja dulu — nggak perlu hapal. Setiap istilah punya analogi sehari-hari biar gampang nyantol.

NodeInti
Analogi: pos kerja di pabrik
Sebuah fungsi yang nerima state, ngerjain sesuatu, lalu balikin perubahan untuk state.
EdgeInti
Analogi: jalur antar pos
Penghubung yang nentuin: setelah node A selesai, lanjut ke node mana?
StateInti
Analogi: tas perjalanan bersama
Tempat penyimpanan data yang dipakai bareng oleh semua node sepanjang graph berjalan.
GraphInti
Analogi: peta keseluruhan
Kumpulan node + edge + state yang dirakit jadi satu alur kerja siap dijalankan.
START / ENDAlur
Analogi: gerbang masuk & keluar
Dua titik bawaan LangGraph yang nandain di mana graph mulai dan selesai.
Conditional EdgeAlur
Analogi: simpang jalan
Edge yang tujuannya ditentukan oleh fungsi — bisa ngelempar ke node berbeda tergantung isi state.
compile()Alur
Analogi: nyetak peta jadi rute
Tahap akhir nge-rakit graph: dari "rancangan" jadi objek graph yang siap dipanggil.
invoke()Alur
Analogi: tekan tombol "jalan"
Cara kita ngejalanin graph dengan state awal sebagai input, lalu nerima state akhir sebagai output.
LLMPendamping
Analogi: rekan kerja yang pintar bahasa
Large Language Model (mis. GPT, Claude). Sering dipanggil di dalam node buat tugas berbahasa.
ToolPendamping
Analogi: perkakas tambahan
Fungsi eksternal (pencarian, kalkulator, API) yang bisa dipakai oleh node atau LLM saat butuh.
Bab 01

Node — pos kerja kecil

Yang paling mengejutkan dari LangGraph: node itu cuma fungsi Python biasa. Nerima state, balikin dictionary perubahan. Itu doang.

  • Bisa nulis fungsi node sederhana yang nerima state dan balikin perubahannya.
  • Tau aturan main output node: dictionary, bukan nilai mentah.

Bayangin node sebagai satu pos kerja di sebuah pabrik kecil. Pos ini nerima paket (state), ngerjain satu hal spesifik, lalu nyerahin paket ke pos berikutnya dengan beberapa info yang udah diupdate.

Aturannya cuma satu: node selalu balikin dictionary. Isinya field-field yang mau di-update di state. Field yang nggak disentuh nggak akan berubah.

node_pertama.py 
# Sebuah node = fungsi yang nerima state, balikin dict perubahan
def sapa(state):
    nama = state["nama"]
    return {"pesan": f"Halo, {nama}!"}

# Coba panggil manual dulu — belum perlu graph
hasil = sapa({"nama": "Rizal"})
print(hasil)
# {'pesan': 'Halo, Rizal!'}
Yang sering bikin bingung

Node nggak ngembaliin state lengkap — cuma bagian yang berubah. LangGraph yang bakal ngegabungin perubahan itu ke state utama. Anggap kayak nyerahin nota perubahan, bukan ngeprint ulang seluruh database.

Misi 01
Level 1 · Mudah ~5 menit

Bikin node "sapa_formal"

Modifikasi fungsi sapa di atas jadi sapa_formal yang menyapa pakai gaya formal: "Selamat pagi, Bapak/Ibu <nama>. Semoga harimu menyenangkan."

Coba panggil manual dengan input {"nama": "Sari"} dan pastikan outputnya benar.

Bonus

Tambahkan field baru di output: "panjang_pesan" berisi jumlah karakter pesan. Berarti satu node bisa balikin lebih dari satu field perubahan.

Petunjuk kalau mentok
Bentuk dasarnya tetap sama: def sapa_formal(state): lalu ambil state["nama"], susun string formalnya, dan return dictionary dengan key "pesan". Buat bonus, tambah satu key lagi: "panjang_pesan": len(pesan).
Bab 02

Edge — penghubung antar pos

Setelah satu node selesai, lanjut ke mana? Edge-lah yang mengatur. Di bab ini kamu mulai merangkai dua node jadi satu alur.

  • Bisa menghubungkan dua node berurutan pakai add_edge.
  • Paham peran START dan END sebagai gerbang masuk dan keluar.

Kalau di bab 1 kita cuma bikin "pos", di sini kita bikin jalannya. Edge nentuin urutan kerja: A dulu, baru B; lalu B selesai, keluar.

LangGraph nyediain dua node spesial: START (titik masuk) dan END (titik keluar). Tugas kita cuma nyambung-nyambungin.

dua_node.py 
from langgraph.graph import StateGraph, START, END
from typing import TypedDict

class State(TypedDict):
    nama: str
    pesan: str

def sapa(state):
    return {"pesan": f"Halo, {state['nama']}!"}

def tutup(state):
    return {"pesan": state["pesan"] + " Sampai jumpa."}

# Rangkai graph
builder = StateGraph(State)
builder.add_node("sapa", sapa)
builder.add_node("tutup", tutup)
builder.add_edge(START, "sapa")     # mulai → sapa
builder.add_edge("sapa", "tutup")   # sapa → tutup
builder.add_edge("tutup", END)     # tutup → selesai

graph = builder.compile()
print(graph.invoke({"nama": "Sari"}))
Output
{'nama': 'Sari', 'pesan': 'Halo, Sari! Sampai jumpa.'}
Misi 02
Level 1 · Mudah ~10 menit

Tambah node "tanggal_hari_ini"

Sisipkan satu node baru di antara sapa dan tutup bernama tanggal_hari_ini yang menambahkan tanggal ke pesan.

Hasil akhirnya kira-kira: "Halo, Sari! Hari ini 27 April 2026. Sampai jumpa."

Bonus

Coba ubah urutannya: tanggal_hari_ini → sapa → tutup. Apa yang terjadi sama hasil pesannya? Kenapa?

Petunjuk kalau mentok
Pakai from datetime import date, lalu date.today().strftime("%d %B %Y"). Jangan lupa: tambahin add_node-nya, lalu update add_edge supaya jalurnya jadi START → sapa → tanggal_hari_ini → tutup → END.
Bab 03

State — tas perjalanan bersama

Node-node nggak saling kirim pesan langsung. Mereka baca dan tulis ke state yang sama. Paham state = paham 60% LangGraph.

  • Bisa mendesain skema state sendiri pakai TypedDict.
  • Bisa baca dari state di satu node dan nge-update field di node lain.

State itu kayak tas perjalanan bersama. Setiap node bisa buka tas, ambil yang dibutuhkan, masukin barang baru, lalu lanjutin perjalanan. Semua node lihat tas yang sama.

Skema state didefinisikan sekali di awal. LangGraph pakai itu buat ngecek field apa aja yang valid dan gimana cara nge-mergenya.

state_main.py 
from langgraph.graph import StateGraph, START, END
from typing import TypedDict

class State(TypedDict):
    kalimat: str
    jumlah_kata: int
    huruf_kapital: str

def hitung_kata(state):
    n = len(state["kalimat"].split())
    return {"jumlah_kata": n}

def jadikan_kapital(state):
    return {"huruf_kapital": state["kalimat"].upper()}

builder = StateGraph(State)
builder.add_node("hitung", hitung_kata)
builder.add_node("kapital", jadikan_kapital)
builder.add_edge(START, "hitung")
builder.add_edge("hitung", "kapital")
builder.add_edge("kapital", END)

graph = builder.compile()
hasil = graph.invoke({"kalimat": "belajar langgraph itu seru"})
print(hasil)
Output
{
  'kalimat': 'belajar langgraph itu seru',
  'jumlah_kata': 4,
  'huruf_kapital': 'BELAJAR LANGGRAPH ITU SERU'
}
Catatan

Field di state itu nggak wajib diisi semua di awal. jumlah_kata dan huruf_kapital bakal terisi otomatis seiring node-node berjalan. Itulah sebabnya state tumbuh sepanjang graph dijalankan.

Misi 03
Level 2 · Sedang ~15 menit

State buat ringkasan teks

Bikin graph dengan state berisi: teks, jumlah_kalimat, kalimat_pertama, dan ringkasan.

Tiga node:

1. pisah_kalimat — hitung jumlah kalimat (split dengan tanda titik) dan ambil yang pertama.
2. buat_ringkasan — gabungkan jadi: "Teks ini punya N kalimat. Kalimat pertama: ..."
3. END

Coba dengan input: "LangGraph itu kerangka kerja agent. Ia memakai konsep graph. Belajarnya juga menyenangkan."

Bonus

Tambah field kalimat_terpanjang dan node yang ngisinya. Pikirin: node ini bergantung sama field apa? Apakah perlu dijalanin sebelum atau sesudah buat_ringkasan?

Petunjuk kalau mentok
Buat pisah_kalimat: kalimat = [k.strip() for k in state["teks"].split(".") if k.strip()], lalu return {"jumlah_kalimat": len(kalimat), "kalimat_pertama": kalimat[0]}. Untuk buat_ringkasan, tinggal pakai f-string yang baca dari kedua field tadi.
Bab 04

Graph — rangkaian utuh

Sekarang semua bahan udah ada. Saatnya merangkai graph lengkap dari nol — tiga langkah, dari input mentah jadi output siap pakai.

  • Bisa merancang pipeline 3+ langkah dari mulai mendesain state, nulis node, sampai compile.
  • Tau urutan tipikal bikin graph LangGraph: state → node → edge → compile → invoke.

Pola yang akan kamu pakai berulang-ulang:

1. Definisikan state — apa aja field yang dipakai bareng?
2. Tulis node — fungsi-fungsi yang ngerjain langkah-langkahnya.
3. Pasang edge — jalur dari START sampai END.
4. compile(), lalu invoke().

Berikut contoh pipeline pendek: nerima nama produk & harga, normalisasi, tambah PPN, lalu format jadi label siap tempel.

label_produk.py 
from langgraph.graph import StateGraph, START, END
from typing import TypedDict

class State(TypedDict):
    nama: str
    harga: int
    harga_final: int
    label: str

def normalisasi(state):
    return {"nama": state["nama"].strip().title()}

def tambah_ppn(state):
    final = int(state["harga"] * 1.11)
    return {"harga_final": final}

def buat_label(state):
    label = f"{state['nama']} — Rp{state['harga_final']:,}"
    return {"label": label}

builder = StateGraph(State)
builder.add_node("normalisasi", normalisasi)
builder.add_node("ppn", tambah_ppn)
builder.add_node("label", buat_label)

builder.add_edge(START, "normalisasi")
builder.add_edge("normalisasi", "ppn")
builder.add_edge("ppn", "label")
builder.add_edge("label", END)

graph = builder.compile()
print(graph.invoke({"nama": "  kopi gula aren  ", "harga": 25000}))
Output
{
  'nama': 'Kopi Gula Aren',
  'harga': 25000,
  'harga_final': 27750,
  'label': 'Kopi Gula Aren — Rp27,750'
}
Misi 04
Level 2 · Sedang ~20 menit

Pipeline review pengguna

Bangun graph dari nol yang nerima review pengguna (string), lalu menghasilkan: jumlah kata, sentimen kasar (positif/negatif/netral berdasarkan kata kunci sederhana), dan ringkasan satu kalimat.

Input: {"review": "Produknya bagus banget, pengirimannya cepat juga."}

Hasil yang diharapkan (kira-kira): jumlah kata terhitung, sentimen positif, ringkasan: "Review 7 kata bersentimen positif."

Bonus

Tambah node highlight_kata_kunci yang ngambil kata-kata penting (mis. "bagus", "cepat", "buruk") dari review dan masukin ke field baru di state.

Petunjuk kalau mentok
Buat sentimen sederhana: bikin dua list kata kunci positif dan negatif. Hitung berapa banyak yang muncul di review. Jika positif > negatif → "positif", sebaliknya "negatif", sama → "netral". Untuk ringkasan, gunakan f-string yang baca jumlah_kata dan sentimen dari state.
Bab 05

Cabang — graph yang bisa milih jalur

Sampai sini semua jalurnya lurus. Padahal kekuatan asli LangGraph muncul saat graph bisa nentuin sendiri mau ke mana — tergantung isi state.

  • Bisa nulis fungsi router yang baca state dan balikin nama node tujuan.
  • Bisa pakai add_conditional_edges buat bikin percabangan logika di graph.

Cara kerjanya sederhana: kita nulis satu fungsi router yang nerima state dan balikin label string. Label itu dipetakan ke node tujuan lewat dictionary.

Contoh kasus: kita ngecek panjang teks. Kalau panjang, lempar ke node "ringkas". Kalau pendek, langsung ke node "selesai".

router.py 
from langgraph.graph import StateGraph, START, END
from typing import TypedDict

class State(TypedDict):
    teks: str
    jumlah_kata: int
    output: str

def hitung(state):
    return {"jumlah_kata": len(state["teks"].split())}

def ringkas(state):
    potongan = " ".join(state["teks"].split()[:5]) + "..."
    return {"output": potongan}

def biarkan(state):
    return {"output": state["teks"]}

# Fungsi router — balikin nama jalur, bukan node langsung
def router(state):
    if state["jumlah_kata"] > 10:
        return "panjang"
    return "pendek"

builder = StateGraph(State)
builder.add_node("hitung", hitung)
builder.add_node("ringkas", ringkas)
builder.add_node("biarkan", biarkan)

builder.add_edge(START, "hitung")
builder.add_conditional_edges("hitung", router, {
    "panjang": "ringkas",
    "pendek": "biarkan",
})
builder.add_edge("ringkas", END)
builder.add_edge("biarkan", END)

graph = builder.compile()
print(graph.invoke({"teks": "halo dunia"}))
print(graph.invoke({"teks": "ini adalah teks panjang yang berisi banyak kata supaya kena cabang ringkas"}))
Output
{'teks': 'halo dunia', 'jumlah_kata': 2, 'output': 'halo dunia'}
{'teks': 'ini adalah teks panjang...', 'jumlah_kata': 12, 'output': 'ini adalah teks panjang yang...'}
Pola yang sering muncul

Banyak kasus agent di dunia nyata cuma kombinasi pola ini berulang: klasifikasi → router → eksekusi spesifik. Kalau kamu udah nyaman sama bab 5, kamu udah punya pondasi buat bikin agent beneran.

Misi 05
Level 3 · Menantang ~25 menit

Klasifikasi pertanyaan pengguna

Bangun graph yang nerima pertanyaan pengguna lalu mengarahkannya ke salah satu dari tiga jalur:

jawab_fakta — kalau pertanyaan dimulai dengan "apa", "siapa", "kapan".
jawab_opini — kalau pertanyaan dimulai dengan "menurut", "bagaimana".
tolak_sopan — selain dua kategori di atas.

Setiap jalur menghasilkan field jawaban yang berisi respons dummy yang sesuai (mis. "Ini pertanyaan fakta — saya akan cari jawaban pasti.").

Bonus

Pisahkan klasifikasi ke node sendiri yang ngisi field kategori di state. Lalu router hanya baca state["kategori"] tanpa logika tambahan. Pola "klasifikator → router" ini sering dipakai di production.

Petunjuk kalau mentok
Untuk versi sederhana, fungsi router-nya bisa langsung lihat awalan kata pertanyaan: q = state["pertanyaan"].lower(), lalu rangkaian if q.startswith(...). Untuk bonus, bikin node klasifikasi yang isinya logika if/elif tadi, dan return {"kategori": "fakta"} dst — lalu router cuma return state["kategori"].
