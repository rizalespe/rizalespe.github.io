LangGraph mungkin terdengar teknis. Tapi konsepnya mirip hal-hal yang kamu temui sehari-hari.

🏭
LangGraph seperti Pabrik Resep

Bayangkan pabrik makanan: ada stasiun kerja (nodes), ada ban berjalan yang menghubungkan (edges), dan ada jadwal kapan harus pindah stasiun (kondisi).

ChatGPT Biasa
Tanya → Jawab. Selesai. Satu kali.
LangGraph
Tanya → Pikir → Cek → Cari Info → Pikir Lagi → Jawab
🎮
Seperti Karakter Game RPG

Di game RPG, karakter punya inventory (state), bisa pakai skill (tools), dan punya alur quest yang bercabang (conditional edges).

Tanpa LangGraph
AI hanya jawab. Tidak ingat. Tidak punya "inventory".
Dengan LangGraph
AI punya memori, alat, dan alur kerja yang bisa berulang.
🍳
Seperti Memasak dengan Chef

Chef punya mise en place (state yang disiapkan), langkah-langkah terstruktur (graph), dan bisa koreksi rasa di tengah proses (loop).

AI Biasa
Langsung tebak semua bahan dari satu instruksi.
LangGraph
Ikuti resep, cicipi, tambah bumbu jika perlu, hidangkan.
🕵️
Seperti Detektif Memecahkan Kasus

Detektif punya catatan kasus (state), investigasi berulang (loop), pakai berbagai alat (tools), dan simpulkan ketika cukup bukti (kondisi stop).

Tanpa Loop
Tebak pelaku dari satu clue. Selesai.
Dengan Loop
Kumpulkan bukti, analisis, cari lagi jika belum yakin.
Konsep Inti

4 Pilar LangGraph

Semua yang perlu kamu pahami bisa dirangkum jadi empat konsep utama. Pahami ini, dan kamu sudah 80% jalan.

01
📦

Node — "Pos Kerja"

Node adalah fungsi Python biasa yang punya satu tugas spesifik. Bayangkan pos kerja di pabrik: ada pos "analisis permintaan", pos "cari info di internet", pos "tulis jawaban". Setiap pos menerima bahan baku (state) dan menghasilkan output baru.

fungsi Python biasa
state_masuk
→
def node_fn()
→
state_baru
02
🔗

Edge — "Jalan Penghubung"

Edge adalah penghubung antar node. Ada dua jenis: edge biasa (selalu lanjut ke node berikutnya) dan conditional edge (memilih jalur berdasarkan kondisi — seperti persimpangan jalan).

koneksi & percabangan
node_A
──always──→
node_B
node_A
──if X──→
node_C
atau node_D
03
🎒

State — "Tas Perjalanan"

State adalah tas yang dibawa dari node ke node. Isinya adalah semua informasi yang dibutuhkan sepanjang proses: pesan, hasil pencarian, hitungan iterasi, dll. Setiap node bisa menambah atau mengubah isi tas ini.

memori & konteks bersama
messages:["Hitung 2+2"]
hasil:4
iterasi:2
selesai:False
04
🗺️

Graph — "Peta Keseluruhan"

Graph adalah keseluruhan peta alur kerja yang menggabungkan semua node dan edge. Kamu define node-nya, sambungkan dengan edge, lalu compile — dan jadilah "otak" yang bisa dijalankan berulang kali dengan input berbeda.

orkestrasi penuh
START
→
node1
→
node2
↩
END
💡 Ingat Ini: LangGraph = Kumpulan Fungsi (nodes) + Penghubung (edges) + Memori Bersama (state) + Peta Alur (graph). Sesederhana itu!
Cara Kerja

Begini cara LangGraph bekerja

Dari pertanyaan pengguna sampai jawaban final, ini tahapannya.

💬
Input
Pengguna kirim pesan / pertanyaan
🧠
Reasoning
LLM berpikir, putuskan langkah berikutnya
🔧
Tool Call
Pakai alat: search, kalkulator, API, dll
🔄
Evaluasi
Sudah cukup? Lanjut atau loop lagi?
✨
Output
Jawaban final dikirim ke pengguna
🔄 Kunci LangGraph: Langkah 2–4 bisa berulang berkali-kali sebelum sampai ke Output! Ini yang membuat LangGraph berbeda dari LLM biasa — ia bisa "berpikir dalam siklus" sampai masalah benar-benar terpecahkan.

graph_pertama.py 
from langgraph.graph import StateGraph, END
from typing import TypedDict

# 1. Definisikan "tas" (State) yang dibawa antar node
class TasPesanku(TypedDict):
    pesan: str
    hasil: str

# 2. Buat node (fungsi kerja)
def node_sapa(state: TasPesanku):
    # Ambil pesan, proses, kembalikan update state
    nama = state["pesan"]
    return {"hasil": f"Halo, {nama}! Selamat belajar LangGraph! 🎉"}

# 3. Buat graph & tambahkan node
builder = StateGraph(TasPesanku)
builder.add_node("sapa", node_sapa)

# 4. Sambungkan START → node → END
builder.set_entry_point("sapa")   # mulai dari sini
builder.add_edge("sapa", END)       # lanjut ke END

# 5. Compile & jalankan!
graph = builder.compile()
hasil = graph.invoke({"pesan": "Budi", "hasil": ""})

print(hasil["hasil"])
# Output: Halo, Budi! Selamat belajar LangGraph! 🎉
1

TypedDict sebagai State

State adalah dictionary bertipe. LangGraph pakai ini untuk tahu field apa saja yang ada di "tas".

2

Node = Fungsi Biasa

Node hanyalah fungsi Python yang terima state dan return dictionary berisi field yang mau diupdate.

3

StateGraph Builder

Builder adalah tempat kamu merakit graph: tambah node, sambungkan edge, lalu compile jadi graph yang bisa dijalankan.

4

invoke()

Panggil graph dengan state awal. Hasilnya adalah state akhir setelah semua node selesai berjalan.

state_dan_nodes.py 
from langgraph.graph import StateGraph, END
from typing import TypedDict, List, Annotated
import operator

# State dengan list yang bisa di-append (pakai Annotated + operator.add)
class AngkaState(TypedDict):
    angka: int
    riwayat: Annotated[List[str], operator.add]  # auto-append!
    selesai: bool

# Node pertama: kalikan dengan 2
def kali_dua(state: AngkaState):
    nilai_baru = state["angka"] * 2
    return {
        "angka": nilai_baru,
        "riwayat": [f"× 2 = {nilai_baru}"]  # akan di-append, bukan replace!
    }

# Node kedua: tambah 10
def tambah_sepuluh(state: AngkaState):
    nilai_baru = state["angka"] + 10
    return {
        "angka": nilai_baru,
        "riwayat": [f"+ 10 = {nilai_baru}"],
        "selesai": True
    }

# Rakit graph
builder = StateGraph(AngkaState)
builder.add_node("kali_dua", kali_dua)
builder.add_node("tambah_sepuluh", tambah_sepuluh)

builder.set_entry_point("kali_dua")
builder.add_edge("kali_dua", "tambah_sepuluh")
builder.add_edge("tambah_sepuluh", END)

graph = builder.compile()
state_akhir = graph.invoke({"angka": 5, "riwayat": ["mulai: 5"], "selesai": False})

print(state_akhir["angka"])    # 20
print(state_akhir["riwayat"])  # ['mulai: 5', '× 2 = 10', '+ 10 = 20']
1

Annotated + operator.add

Saat kamu return list dari node, defaultnya mengganti. Dengan Annotated[List, operator.add], ia akan menambahkan ke list yang ada.

2

Multi-node Pipeline

Dua node berjalan berurutan. Node pertama mengubah state, node kedua meneruskan dari state yang sudah diubah.

3

State sebagai "audit trail"

Field riwayat merekam setiap langkah. Ini berguna untuk debugging dan memahami apa yang terjadi di dalam graph.

4

State tidak di-reset

Nilai dari node pertama tetap ada saat node kedua berjalan. Inilah "memori bersama" antar node.

conditional_edge.py 
from langgraph.graph import StateGraph, END
from typing import TypedDict

class HitungState(TypedDict):
    angka: int
    iterasi: int
    pesan: str

# Node: tambah terus sampai > 100
def tambah_terus(state: HitungState):
    return {
        "angka": state["angka"] + 17,
        "iterasi": state["iterasi"] + 1
    }

# 🔑 Fungsi kondisi: ini yang memutuskan "mau ke mana?"
def cek_cukup(state: HitungState) -> str:
    if state["angka"] > 100:
        return "selesai"   # → node "selesai"
    else:
        return "lanjut"    # → balik ke "tambah_terus" (LOOP!)

# Node penutup
def node_selesai(state: HitungState):
    return {"pesan": f"Selesai! Nilai {state['angka']} setelah {state['iterasi']} iterasi"}

# Rakit
builder = StateGraph(HitungState)
builder.add_node("tambah_terus", tambah_terus)
builder.add_node("selesai", node_selesai)

builder.set_entry_point("tambah_terus")

# add_conditional_edges: dari node mana, fungsi kondisi, peta hasil
builder.add_conditional_edges(
    "tambah_terus",   # dari node ini
    cek_cukup,         # jalankan fungsi ini
    {                  # map: hasil fungsi → node tujuan
        "selesai": "selesai",
        "lanjut": "tambah_terus"   # LOOP ke diri sendiri!
    }
)
builder.add_edge("selesai", END)

graph = builder.compile()
hasil = graph.invoke({"angka": 0, "iterasi": 0, "pesan": ""})
print(hasil["pesan"])
# Output: Selesai! Nilai 102 setelah 6 iterasi
1

Conditional Edge = Persimpangan

Fungsi kondisi mengembalikan string, dan string itu di-map ke node tujuan. Seperti lampu hijau/merah di persimpangan.

2

Loop = Kembali ke diri sendiri

Ketika edge kembali ke node yang sama, terjadi loop. Ini inti kekuatan LangGraph: bisa iterasi sampai kondisi terpenuhi.

3

Cegah infinite loop

Selalu pastikan ada kondisi yang akan menghentikan loop. Gunakan counter iterasi sebagai safety net.

4

Pola ReAct

Think → Act → Observe → Think lagi. Conditional edge yang loop inilah yang memungkinkan pola agent ReAct.

agent_dengan_tools.py 
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool
from langgraph.graph import StateGraph, END
from langgraph.prebuilt import ToolNode
from langchain_core.messages import HumanMessage
from typing import TypedDict, Annotated, List
import operator

# Definisi tools (alat yang bisa dipakai agent)
@tool
def kalkulator(ekspresi: str) -> str:
    """Hitung ekspresi matematika. Contoh: '2 + 3 * 4'"""
    try:
        return str(eval(ekspresi))
    except:
        return "Error: ekspresi tidak valid"

@tool
def cek_ganjil_genap(angka: int) -> str:
    """Cek apakah angka ganjil atau genap"""
    return "genap" if angka % 2 == 0 else "ganjil"

tools = [kalkulator, cek_ganjil_genap]

# LLM yang tahu cara pakai tools
llm = ChatOpenAI(model="gpt-4o-mini")
llm_with_tools = llm.bind_tools(tools)

# State pakai Messages (standar untuk agent)
class AgentState(TypedDict):
    messages: Annotated[List, operator.add]

# Node agent: LLM memutuskan apa yang harus dilakukan
def agent(state: AgentState):
    respons = llm_with_tools.invoke(state["messages"])
    return {"messages": [respons]}

# Kondisi: lanjut ke tools atau selesai?
def harus_pakai_tool(state: AgentState) -> str:
    last_msg = state["messages"][-1]
    if hasattr(last_msg, "tool_calls") and last_msg.tool_calls:
        return "pakai_tool"
    return "selesai"

# Rakit graph
builder = StateGraph(AgentState)
builder.add_node("agent", agent)
builder.add_node("tools", ToolNode(tools))  # prebuilt!

builder.set_entry_point("agent")
builder.add_conditional_edges("agent", harus_pakai_tool,
    {"pakai_tool": "tools", "selesai": END})
builder.add_edge("tools", "agent")  # setelah pakai tool, balik ke agent

graph = builder.compile()
hasil = graph.invoke({
    "messages": [HumanMessage("Hitung 15 * 7, lalu cek apakah hasilnya ganjil atau genap")]
})
print(hasil["messages"][-1].content)
# Agent akan pakai kalkulator, lalu cek_ganjil_genap, lalu jawab
1

@tool decorator

Ubah fungsi Python biasa jadi "alat" yang bisa digunakan LLM. Docstring-nya penting — itu yang dibaca LLM untuk tahu kapan memakai alat ini.

2

bind_tools()

Hubungkan tools ke LLM. Sekarang LLM tahu ada alat apa saja yang tersedia dan bisa meminta untuk menggunakannya.

3

ToolNode (prebuilt)

LangGraph sudah menyediakan ToolNode siap pakai. Ia otomatis mengeksekusi tool yang diminta LLM dan mengembalikan hasilnya.

4

Pola ReAct Loop

agent → (jika perlu tool) → tools → agent → (jika perlu tool lagi) → tools → ... → END. Ini pola standar semua AI agent!

Pro Tips

Hal penting yang sering terlupakan

🛡️

Selalu ada kondisi stop di loop

Setiap loop harus punya exit condition. Tambahkan counter max_iterations sebagai safety net agar loop tidak jalan selamanya.

📝

Docstring tool itu krusial

LLM membaca docstring untuk tahu kapan dan bagaimana pakai tool. Buat docstring yang jelas: apa fungsinya, kapan dipakai, contoh inputnya.

🔍

Visualisasikan graph kamu

Gunakan graph.get_graph().draw_mermaid_png() untuk melihat diagram graph kamu. Sangat membantu untuk debugging alur!

🗂️

State harus serializable

Isi state harus bisa di-serialize (JSON-compatible) jika kamu pakai checkpointer untuk menyimpan state ke database.

⚡

Gunakan prebuilt jika bisa

LangGraph punya ToolNode, create_react_agent, dan lainnya. Mulai dari sini sebelum buat custom dari nol.

🧪

Test node secara individual

Karena node hanyalah fungsi Python, kamu bisa test setiap node secara independen dengan memanggil langsung tanpa perlu compile graph.

Kamus Istilah

Glosarium LangGraph

Istilah-istilah yang akan sering kamu temui, dengan penjelasan simpel dan analogi sehari-hari.

Node
Fungsi Python yang melakukan satu tugas spesifik dalam graph. Menerima state, mengembalikan state yang diupdate.
🏭 Seperti: pos kerja di pabrik
Edge
Koneksi yang menghubungkan satu node ke node lainnya. Menentukan urutan eksekusi.
🛣️ Seperti: jalan yang menghubungkan antar kota
State
Data bersama yang dibawa dan dimodifikasi saat graph berjalan dari node ke node.
🎒 Seperti: tas yang kamu bawa dan isi selama perjalanan
Conditional Edge
Edge yang memilih tujuan berdasarkan kondisi tertentu. Hasil fungsi kondisi menentukan node mana yang dipanggil.
🚦 Seperti: persimpangan dengan rambu lalu lintas
Graph
Keseluruhan alur kerja yang terdiri dari node-node yang terhubung dengan edge. Setelah compile, bisa dijalankan.
🗺️ Seperti: peta perjalanan lengkap dari A ke Z
Tool
Fungsi Python yang bisa dipanggil oleh LLM. Dibuat dengan @tool decorator dan docstring yang jelas.
🔧 Seperti: peralatan dalam kotak perkakas mekanik
Agent
LLM yang diberi tools dan loop, sehingga bisa memutuskan sendiri langkah apa yang perlu diambil untuk menyelesaikan tugas.
🕵️ Seperti: detektif yang investigasi mandiri
invoke()
Metode untuk menjalankan graph secara sinkronus dengan state awal. Mengembalikan state akhir setelah semua node selesai.
▶️ Seperti: tombol play yang menjalankan semuanya
Checkpointer
Komponen untuk menyimpan state ke storage (SQLite, Redis, dll) agar graph bisa dilanjutkan nanti atau di-debug.
💾 Seperti: fitur save game di video game
ReAct Pattern
Pola agent: Reason → Act → Observe → Reason lagi. Loop antara LLM (reasoning) dan tools (acting) sampai tugas selesai.
🔄 Seperti: siklus PDCA dalam manajemen
TypedDict
Cara mendefinisikan struktur state di Python dengan type hints. Membuat state lebih terstruktur dan mudah di-debug.
📋 Seperti: formulir dengan kolom-kolom yang sudah ditentukan
Annotated + operator.add
Cara membuat field di state yang di-append (bukan di-replace) saat node mengembalikan nilai baru.
📜 Seperti: buku catatan yang selalu ditambahkan, tidak dihapus
